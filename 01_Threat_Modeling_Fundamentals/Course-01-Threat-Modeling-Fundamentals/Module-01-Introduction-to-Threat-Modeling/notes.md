- Threat modeling = finding and fixing security risks early

- Focus on security during design phase

- Main phases:
  - Design
  - Break
  - Fix
  - Verify

- Step 1: Design
  - Understand system structure
  - Identify components and data flow

- Step 2: Break
  - Think like an attacker
  - Find possible vulnerabilities

- Step 3: Fix
  - Apply security controls
  - Reduce risks

- Step 4: Verify
  - Check if fixes work
  - Validate security improvements

- Continuous process, not one-time
