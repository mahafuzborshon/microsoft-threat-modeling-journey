- Use frameworks to identify threats systematically

- STRIDE model:

  - Spoofing:
    - Pretending to be someone else

  - Tampering:
    - Modifying data without permission

  - Repudiation:
    - Denying actions performed

  - Information Disclosure:
    - Accessing sensitive data

  - Denial of Service (DoS):
    - Overloading system

  - Elevation of Privilege:
    - Gaining higher access rights

- Benefits:
  - Structured thinking
  - Covers multiple threat types
  - Reduces chances of missing risks

- Key idea:
  - Use STRIDE to analyze each system component
