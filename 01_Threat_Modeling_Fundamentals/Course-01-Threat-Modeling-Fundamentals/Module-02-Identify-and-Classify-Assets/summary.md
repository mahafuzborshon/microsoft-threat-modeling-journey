In this module, I learned how to create a threat model using data flow diagrams (DFD). This helped me visualize how a system works and how different parts interact with each other.

I understood that breaking a system into elements like processes, data stores, and data flows makes it easier to analyze security risks. It gave me a clearer picture of how data moves and where vulnerabilities might exist.

The concept of trust boundaries was especially important. It showed me where the level of trust changes, which is where most security risks appear.

Overall, this module made threat modeling more practical by using diagrams instead of just thinking abstractly.
