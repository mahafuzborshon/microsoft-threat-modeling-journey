- Data Flow Diagram (DFD) = visual representation of a system

- Purpose:
  - Understand system structure
  - Identify how data moves
  - Detect potential security risks

- DFD Elements:

  - Process:
    - Represents operations or tasks
    - Where data is processed

  - Data Store:
    - Storage of data (database, files)
  
  - External Entity:
    - Outside system (users, external systems)
    - No control over it

  - Data Flow:
    - Movement of data between elements

  - Trust Boundary:
    - Where trust level changes
    - High-risk areas

- Key idea:
  - Visualizing system = easier threat identification

- Important:
  - Focus on how data flows, not just components
