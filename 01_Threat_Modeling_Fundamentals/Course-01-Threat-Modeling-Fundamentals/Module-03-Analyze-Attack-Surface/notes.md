- Depth layer = level of detail in a diagram

- Importance:
  - Too simple → miss threats
  - Too complex → hard to understand

- Use layered approach

- Layers in DFD:

  - Layer 0 (System level):
    - High-level overview
    - Entire system as one unit

  - Layer 1 (Process level):
    - Break system into main processes

  - Layer 2 (Subprocess level):
    - More detailed breakdown

  - Layer 3 (Lower-level):
    - Deep technical details

- Goal:
  - Provide enough context for analysis
  - Keep it understandable

- Key idea:
  - Balance between simplicity and detail
